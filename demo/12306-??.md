# Demo: 在 12306 上引导用户退票

> 评委一看就懂的强痛点场景。**12306 退票入口埋得很深**，第一次退票的人 90% 找不到。

## 场景设定

用户是第一次在 12306 退票的大学生。已经登录、有一张未出行的票。打开 PointMe 问：

> "我想退一张明天去合肥的票，怎么操作？"

## 期望 agent 行为

| 步 | agent 调用 | 用户看到 |
|---|---|---|
| 1 | `observe()` 拿当前页（很可能是首页） | 思考中 |
| 2 | `clear_overlay` + `highlight(顶栏"我的12306")` + `annotate("先去这里查订单")` + `wait_for_user_action({type:"click", refId})` | 顶栏一个橙色脉冲圆环 + 气泡 |
| 3 | 用户点击 → 触发 `urlchange` | — |
| 4 | `observe` 抓新页 → 高亮"未出行订单" + annotate | 第二步指引 |
| 5 | 用户点进订单详情 → 高亮"退票"按钮 | 第三步指引 |
| 6 | 用户进入退票确认页 → `done("退票确认页已经打开，按页面提示完成即可。")` | overlay 清空 + 总结 |

## 路演彩蛋

agent 可以在第 2 步主动 `ask_user("你的订单是用哪个手机号注册的？需要选择对应账号")` —— 这是体现 "agent 主动反问、不瞎猜" 的好桥段。

## 易踩雷

- 12306 用大量 jQuery 老页面，部分按钮没 ARIA。snapshot 里 `name` 字段会从 `innerText` 兜底，应该 OK。
- 退票二次确认页有 iframe — content script 默认不进 iframe，需要时给 manifest 加 `"all_frames": true`。
