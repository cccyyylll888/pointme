# PointMe — 3 分钟路演稿

## 钩子（30s）

> 各位评委，我先问一个问题：你妈妈在 12306 上能找到"退票"按钮吗？
>
> 90% 的复杂网站，新用户的痛苦不是"看不懂内容"，而是**"找不到那个按钮"**。
>
> 现在的 AI agent 都在卷一件事：**替你点**。Claude for Chrome、Browser Use、Computer Use —— 它们都在抢"代操作"赛道。
>
> 我们做的是反过来的事：**站在你身边，指着屏幕教你自己点**。

## 演示（90s）

打开 12306 主页 → 召唤右下角小 🦊 → 输入"我想退明天去合肥那张票"

观众看到屏幕上：
1. 顶栏出现一圈橙色脉冲光环 + 气泡 "先去这里查订单"
2. 主持人点了，agent 等了 0.5 秒，进入新页面
3. 又一个高亮 + 新气泡引导到下一步
4. 一路点到退票确认页 → 小狐狸说"已经到这一步了，剩下的按页面提示就好"

> 注意：**全程没有任何一次代用户点击**。Agent 是教练，用户是球员。

## 为什么是现在（30s）

- Claude Sonnet 4.6 多模态够便宜，能秒级理解 a11y 树
- prompt cache 让连续多轮 a11y 树几乎免费
- MV3 + a11y API 第一次让浏览器扩展拿得到稳定的语义结构

三件事任缺一件，三年前都做不出来。

## 商业化（30s）

- **B2C**：装机工具，免费 + 月度订阅高级模型
- **B2B SDK**：站长一行 `<script>` 嵌入，按 query 计费 —— 同时拿到**用户问什么**的画像数据
- **企业内训**：复杂 SaaS（SAP、Salesforce、内部系统）的新员工引导，替代死板的 product tour

国内市场尤其值得做：政务/银行/教务/医疗系统的复杂度天花板比海外高一个量级。

## 评委可能问的问题

**Q1: 和 Pendo / Appcues 这种 product tour 工具有什么不同？**
A: 那些是站长**预先脚本化**的。每个 tour 都得运营手动配。我们是**用户问什么 agent 即时生成**，覆盖任意任务、任意页面。

**Q2: 和 Claude for Chrome 有什么不同？**
A: Claude for Chrome **代你点**，它越做越好用户越被动。我们**教你点**，越用用户越熟练。这是哲学差异，不是技术差异。

**Q3: 怎么处理隐私？a11y 树会被发到云上模型？**
A: 是的，所以扩展支持企业自部署 proxy + 自带模型 endpoint。本地没存任何用户数据。
